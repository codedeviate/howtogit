# howtogit

Two in-depth, best-practices reference books for **git** and **gh**, written in Markdown and rendered to PDF using [recon](https://github.com/codedeviate/recon). Each book covers fundamental concepts, everyday workflows, best practices, pitfalls, and real-world examples ‚Äî with practical, copy-pasteable commands and clear explanations of the underlying mechanics.

## What's inside

### Git book

A comprehensive guide to distributed version control, organized in eight parts:

- **Part I**: Intro & Setup ‚Äî What git is and how to install it
- **Part II**: Creating & snapshotting ‚Äî `init`, `clone`, `add`, `status`, `diff`, `commit`, `restore`, `reset`, `rm`, `mv`, `clean`, `stash`
- **Part III**: Branching, merging, history rewriting ‚Äî `branch`, `checkout`, `switch`, `merge`, `mergetool`, `rebase`, `cherry-pick`, `revert`, `tag`, `bisect`, `rerere`
- **Part IV**: Inspecting & comparing ‚Äî `log`, `show`, `blame`, `shortlog`, `describe`, `reflog`, `grep`, `range-diff`
- **Part V**: Sharing & remotes ‚Äî `remote`, `fetch`, `pull`, `push`, `submodule`, `worktree`
- **Part VI**: Patching & email workflow ‚Äî `apply`, `am`, `format-patch`, `send-email`, `request-pull`
- **Part VII**: Administration ‚Äî `gc`, `fsck`, `maintenance`, `archive`, `bundle`, `notes`, `sparse-checkout`, `replace`, `filter-branch`, `hooks`
- **Part VIII**: Troubleshooting ‚Äî Getting out of jams (organized by symptom, not command)

### GitHub CLI (gh) book

A practical guide to GitHub from the command line, organized in eight parts:

- **Part I**: Intro & Auth ‚Äî What gh is and how to authenticate
- **Part II**: Repositories & code ‚Äî `repo`, `browse`, `search`
- **Part III**: Collaboration ‚Äî `pr`, `issue`, `label`, `project`
- **Part IV**: CI/CD & automation ‚Äî `run`, `workflow`, `cache`
- **Part V**: Releases & artifacts ‚Äî `release`, `gist`, `attestation`
- **Part VI**: Configuration & secrets ‚Äî `secret`, `variable`, `ruleset`, `ssh-key`, `gpg-key`, `org`
- **Part VII**: Extending gh & scripting ‚Äî `api`, `extension`, `status`, `completion`
- **Part VIII**: Troubleshooting ‚Äî Getting out of jams (organized by symptom, not command)

## Prerequisites

To build the PDFs, you need:

- **recon** ([github.com/codedeviate/recon](https://github.com/codedeviate/recon)) ‚Äî the Markdown-to-PDF build engine
- **agent-browser** ‚Äî command-line browser driver (on PATH)
- **Chrome** ‚Äî required by recon's PDF rendering

## Building the PDFs

Prebuilt PDFs are committed in [`dist/`](dist/) ‚Äî grab [`dist/git.pdf`](dist/git.pdf)
or [`dist/gh.pdf`](dist/gh.pdf) directly if you just want to read. To rebuild
them from source:

```sh
make all       # Build both git.pdf and gh.pdf ‚Üí dist/
make git       # Build only the git book
make gh        # Build only the gh book
make lint      # Check chapter structure and linting rules
make clean     # Remove the dist/ directory
```

## Reading the source

- Chapters live in `book/git/` (git book) and `book/gh/` (gh book)
- Read `book/<book>/order.txt` to see the chapter order and part structure
- Contribution and authoring rules are in [`STYLE.md`](STYLE.md) ‚Äî mandatory for all chapters
  - One H1 per chapter (the title)
  - Standard command-chapter sections: Mental model, Synopsis, Everyday usage, Key options, Best practices, Pitfalls & gotchas, Worked examples, Recovery, See also
  - Troubleshooting chapters organized by symptom, not command
  - All code examples must be realistic and verified against `--help`
